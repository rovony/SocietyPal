Why composer install (not update)?

Using install ensures reproducible builds by using exact versions from composer.lock. The update command would change package versions and potentially break functionality.

Rollback Script
Prepare automated rollback procedure
bash
#!/bin/bash # rollback.sh - Emergency rollback script echo "🔄 Starting emergency rollback..." # Switch to previous release cd /var/www/societypal.com PREVIOUS_RELEASE=$(ls -t releases/ | sed -n '2p') if [ -z "$PREVIOUS_RELEASE" ]; then echo "❌ No previous release found" exit 1 fi echo "📦 Rolling back to: $PREVIOUS_RELEASE" ln -nfs releases/$PREVIOUS_RELEASE current echo "✅ Rollback completed successfully" echo "🌐 Application restored to previous version"

Test Rollback: Always test rollback procedures in staging environment
